# 表格分镜导入指南

## 核心原则

输入剧本已经是 Markdown 分镜表格时，必须按表格逐行导入结构化分镜，不要重新拆解、扩写、合并、删减或重排镜头。

支持表头格式：

```markdown
| 镜头号 | 景别 | 时长 | 画面描述 | 台词/独白 |
|---|---|---|---|---|
```

## 使用步骤

1. 调用 `read_storyboard_context` 读取 `script`、角色、场景、已有分镜。
2. 从 `script` 中解析 Markdown 表格行。
3. 每一行创建一个 storyboard，`shot_number` 必须等于“镜头号”。
4. 调用 `save_storyboards` 一次性保存所有镜头。

## 字段映射

- `shot_number`：使用“镜头号”的数字。
- `shot_type`：使用“景别”，保留原词，如 `特写 / 近景 / 中景 / 全景 / 主观 / 慢镜 / 黑屏`。
- `duration`：从“时长”提取秒数，`3s` 保存为 `3`。
- `description`：使用“画面描述”原文。
- `action`：使用“画面描述”原文，不额外扩写。
- `dialogue`：使用“台词/独白”原文；如果是 `（无）`、`无`、空白，则保存为空。
- `title`：用画面描述提炼 3-8 个中文字的短标题；只概括，不新增剧情。
- `result`：可留空；不要为了填字段编造动作后果。
- `bgm_prompt`、`sound_effect`：留空，除非原表格明确提供。

## 可补充字段

只允许在已有表格信息基础上做低风险补充：

- `scene_id`：优先根据 `read_storyboard_context.scenes` 的地点/时间匹配；无法确定则留空。
- `location`：如果画面描述或台词里明确出现地点，提取；否则可用匹配到的场景地点；仍不确定则留空。
- `time`：如果画面描述明确出现早晨、夜晚、黄昏等，提取；否则留空。
- `character_ids`：只绑定画面描述或台词中明确出现、说话、被看见的人物；从 `read_storyboard_context.characters` 选择 ID，不要编造角色。
- 说话人判断优先级：`台词/独白` 单元格的显式角色名 > 明确的系统音/旁白 > 已绑定角色。不要因为画面里出现 A，就把 `（B独白说）：...` 改成 A。
- `atmosphere`：只从画面描述中提取明确氛围词，如暧昧、压抑、冷峻、晨雾、昏暗；不要写音乐或音效。
- `angle`、`movement`：只有表格内容明确体现时才填写；否则留空或用 `固定`，不要虚构复杂运镜。

## 图片提示词 image_prompt

`image_prompt` 是镜头第一帧图片提示词，必须前瞻性地描述“动作开始前的一瞬间”，不要描述完整动作过程。

生成规则：

- 只基于“画面描述”提取第一帧画面。
- 必须写成静态图片构图：谁在画面里、身体姿态、朝向、视线方向、空间位置、光线和氛围。不要写“正在远去、继续走、马蹄声、看着某人离去”这类过程。
- 如果画面描述是连续动作，只写动作即将开始、刚进入状态或已经摆好的第一帧姿态。
- 不要写后续动作结果、运动完成状态、情绪转折后的状态。
- 不要加入 `video_prompt` 的分段内容。
- 不要加入英文 prompt、画质词、风格词、技术参数。
- 如果画面描述本身就是静态表情/环境，直接改写成单帧画面即可。

### 首帧主体可见原则

- 视频模型只能参考首帧中已经出现的主体。凡是本镜头接下来要运动、说话、被运镜表现的人物，首帧里必须可见，且要写清楚身份、姿态和朝向。
- 除非本镜头明确是纯器官/道具特写，否则不要只写“一只手”“一双眼睛”“靴子”等孤立部位。需要写成 `西门庆的手连同深蓝衣袖出现在牌匾前`、`潘金莲半张脸和手指同时在窗缝后出现` 等，让角色身份和身体上下文成立。
- 如果表格画面确实只表现一个身体器官或道具，`image_prompt` 可以只写该局部，但必须保留可识别的归属线索，如衣袖、服饰、手中道具、相邻环境；不要让视频模型误以为后续会凭空出现完整人物。
- 不要在首帧里提及画外人物名字。比如 `武大郎看着西门庆离去的方向`，首帧应写 `武大郎瘫在泥水中，脸偏向街道深处或画面远处`，不要写 `西门庆离去的方向`，除非西门庆确实还在画面里并需要以参考人物出现。
- 离场类镜头必须把离场人物仍然放在首帧中。比如 `西门庆扬鞭远去`，首帧应写 `西门庆骑在马上，背对镜头，身体微微前倾，马头朝向街道深处，晨雾包围背影`，不要只写“背影远去”。
- 视线方向用空间词表达：`看向街道深处`、`偏头望向画面右侧`、`目光落向门口方向`。只有当目标人物在画面内且需要生成时，才写目标人物名字。
- 黑屏镜头的 `image_prompt` 只写 `画面全黑`，不要补人物、环境或悬念描述。

示例：

- 表格画面：`一只修长的手，指尖轻轻划过牌匾上斑驳的金漆。`
- `image_prompt`：`西门庆的手连同深蓝长袍衣袖出现在庆丰堂牌匾前，指尖悬在斑驳金漆上方即将触碰，晨光贴着木纹，画面压抑克制。`

- 表格画面：`西门庆继续看向前方，眼神中透着掌控一切的傲慢。`
- `image_prompt`：`西门庆侧身站在街边，目光已经投向前方，嘴角微压，晨雾中的光线落在深蓝长袍肩头。`

- 表格画面：`西门庆扬鞭，马蹄声哒哒远去，背影挺拔融入晨雾。`
- `image_prompt`：`西门庆骑在马上，背对镜头立在街道中央，右手握着马鞭，马头朝向街道深处，深蓝长袍和挺拔背影被晨雾包围。`

- 表格画面：`武大郎瘫软在泥水中，不敢起身，看着西门庆离去的方向。`
- `image_prompt`：`武大郎瘫软在泥水中，双手撑在湿冷石板上，身体蜷缩，脸偏向街道深处，眼神惊惧卑微；画面里只保留武大郎和空街晨雾。`

## 视频提示词 video_prompt

`video_prompt` 用来表现这一行镜头在对应时长内的运动过程。不要重新拆成多个镜头，只把本行镜头按时间推进描述。

格式：

```text
0-3秒：<location>地点</location>，景别，<role>角色</role>动作开始。
<n>3-6秒：<location>地点</location>，景别，动作继续或表情变化。
```

规则：

- 总时长小于等于 3 秒时，只写一个 `0-X秒` 段。
- 超过 3 秒时，按 3 秒左右切段，最后一段可不足 3 秒。
- 使用表格中的“景别”和“画面描述”。
- 出现角色名时用 `<role>角色名</role>` 标注。
- 出现地点时用 `<location>地点</location>` 标注。
- 独白或旁白可用 `<voice>角色名</voice>` 标注，但不要写音效或音乐。
- 不要加入英文画质、风格、技术参数。

## 台词/独白处理

- `dialogue` 的说话人只能来自“台词/独白”单元格本身，不允许根据“画面描述”里的可见人物改写说话人。
- `角色：内容` 保存为 `角色：内容`，如果出现 `角色：：内容` 或 `角色::内容`，必须清理为单个冒号。
- `（角色名独白说）：内容`、`（角色名独白）：内容`、`（角色名内心OS）：内容` 都必须识别为“角色名”的独白，保存为 `角色名：内容`；不要保存成 `角色名独白说：内容`，也不要丢掉角色名。
- 独白即使画面主体是另一个人，也必须以括号内的角色名为准。例如画面描述是武大郎，台词是 `（西门庆独白说）：看那个矮子。`，`dialogue` 必须保存为 `西门庆：看那个矮子。`。
- 如果“台词/独白”单元格以冒号开头，如 `：内容`，说明说话人丢失，必须回看原表格上下文；无法确定时不要绑定到画面人物，宁可保存为空并在 `description` 保留画面信息。
- `（无）` 不保存。
- 不要把台词改写进 `image_prompt`。
- `video_prompt` 可表现角色说话的口型或神态，但不要复述过长台词。

## 禁止事项

- 禁止把 25-30 行表格重新拆成新的镜头。
- 禁止合并连续表格行。
- 禁止把一个镜头拆成多个 storyboard。
- 禁止在 `image_prompt` 中描述完整动作过程。
- 禁止把 `video_prompt` 拼进 `image_prompt`。
- 禁止输出解释性文字代替调用 `save_storyboards`。
